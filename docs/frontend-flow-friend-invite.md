# 好友与群邀请 · 前端交互流程稿

适用场景：**小程序内「好友」≠ 微信通讯录好友**。邀请链接无法像微信私聊那样「点对点发给某个联系人」，需通过 **分享 / 复制码 / 扫码 / 站内通知** 等路径触达对方。

---

## 一、角色与前提

| 角色 | 说明 |
|------|------|
| A（发起方） | 已登录小程序，可能已完成 DNA |
| B（对方） | 可能被推荐、被加为「小程序好友」，**不一定是 A 的微信好友** |

**共识**：邀请的本质是让对方带着 **`invite_code`**（或等价参数）打开小程序并调用 `POST /api/groups/join-by-invite`。

---

## 二、流程 A：首页推荐 · 点心加好友

```text
[首页-推荐卡片]
    │
    ├─ 点 ❤  ──► 调用 POST /api/friends/add { target_user_id }
    │             成功 → Toast「已添加到好友」
    │             幂等已有 → Toast「已在好友列表」
    │
    └─ 点 ✕  ──► 前端本地跳过该条（可选上报埋点）；列表看下一条
```

**好友 Tab**

```text
[Tab-好友]
    │
    └─ 进入页 ──► GET /api/friends，展示列表（昵称、头像、标签）
                  点击某项 ──► 可选：跳转「好友资料」或占位「发邀请」入口
```

**文案建议**

- 点心后：**「已加入小程序好友（非微信好友）」** 一行小字说明，避免用户误以为已加微信。

---

## 三、流程 B：发起建群 · 获取邀请信息

```text
[发起建群]
    │
    ├─ 填写群名、城市、人数等 ──► POST /api/groups
    │                             成功返回 group_id、invite_code（若后端已返回）
    │
    └─ 进入「群详情 / 群主工具区」
            │
            ├─ 若需完整分享信息 ──► GET /api/groups/{group_id}/invite（仅群主）
            │                       得到 invite_code、invite_path（如 /pages/group/join?code=xxx）
            │
            └─ 展示「邀请好友」面板（见下一节）
```

---

## 四、流程 C：把邀请给对方（产品内好友，非微信好友）

以下 **并行展示**，用户任选一种即可。

### 必读：微信里「不能直接私聊发给 B」，那 B 到底是怎么收到的？

先要统一认知（避免产品和用户误解）：

| 事实 | 说明 |
|------|------|
| ① | **小程序不能像微信好友那样**，给「非通讯录好友」的某个人 **单独强塞一条微信私聊**。微信 **没有** 开放这种接口。 |
| ② | **「分享给微信」** 时，用户 A 实际是在微信里 **选一个发送对象**：可以是 **某个微信群**、**文件传输助手**、**某个已是好友的人**，不是「输入对方小程序昵称就自动送达」。 |
| ③ | 因此 **「发给 B」** 在工程上 =：让 B **在某处拿到同一张卡片 / 同一段码 / 同一个二维码**，B 再 **自己点开小程序**。这个「某处」一定是下面三类之一：**微信内的群/朋友圈等**、**微信外的渠道**、**你们自己做站内通知**。 |

**和「小程序内加了好友」的关系**

- 小程序里的「好友」只是你们库里的关系，**不会自动变成微信会话里的一条消息**。
- 若 A、B **不是微信好友**，又 **不在同一个微信群**，则 **仅靠微信分享无法保证「只给 B」或「必然被 B 看到」**。这时必须走 **复制码 + 别的渠道**，或二期做 **站内邀请通知**。

---

### C1 分享小程序卡片（优先）——「发给谁」在微信里怎么选？

```text
[邀请好友] 面板
    │
    └─「分享到微信」──► 使用小程序分享能力（onShareAppMessage）
        - sharePath 带 invite_code 或 group 参数，与后端约定一致
        - 说明文案：「对方无需是你的微信好友，点开链接即可加入（需先登录小程序）」
```

**B 能收到的典型情况（不需要 A、B 是微信好友）**

1. **两人同在某个微信群**（驴友群、校友群、豆瓣同城群对应的微信群等）  
   - A 点分享 → 选择 **该微信群** → 卡片出现在群里 → **B 在群里点开** → 落地页带 `code` → `join-by-invite`。  
   - 这是 MVP **最现实、最常用** 的「触达非好友 B」的方式。

2. **A 发朋友圈 / 状态（若你们小程序分享支持且运营允许）**  
   - B 刷到后点开（B 同样不必是 A 的好友，取决于朋友圈可见范围）。

3. **A 先发到「文件传输助手」再转发**  
   - 仅方便 A 自己；若要让 B 收到，仍需 A 再转发到 **群** 或通过 **别的渠道** 把链接给 B。

**B 收不到的典型情况**

- A、B **无共同微信群**、**不是好友**、B **不看朋友圈** → 仅靠「分享到微信」**无法保证** B 收到，必须配合 **C2 复制码** 或 **站内通知（流程 E）**。

---

### C2 复制邀请码 / 短文案——「发给 B」的载体是微信以外的通道

```text
[复制邀请码] ──► 写入剪贴板：invite_code 或「一行说明 + 小程序搜索关键词」
                  Toast「已复制，发给对方即可」
```

**这里的「发给」不是系统代发，而是用户 A 自己粘贴到：**

| 载体 | 举例 |
|------|------|
| 短信 / iMessage | A 把码或短链发给 B 的手机号 |
| 钉钉 / QQ / 飞书 / 邮件 | 同一逻辑 |
| 小红书 / 抖音私信 | 若两人在那边认识 |
| 线下 | 当面口述码、或发截图 |

对方仍需要在小程序 **落地页**（配置好路径）里带 `code`，`onLoad` 读 `options.code` → `POST /api/groups/join-by-invite`。

---

### C3 二维码（可选）——载体是「当面」或「发图片」

```text
[保存群二维码] ──► 后端生成小程序码（需额外接口/运维配置）
                  当面扫码加入，适合线下碰面
```

**B 怎么收到**

- **当面**：B 扫 A 的手机屏幕。  
- **远程**：A 把 **二维码图片** 发到 **微信群 / 短信彩信 / 其它 IM**（逻辑同 C2，只是载体变成图片）。

---

### 小结：三条路径和「怎么发给 B」的对应关系

| 路径 | 本质 | B 收到邀请的常见方式 |
|------|------|----------------------|
| 分享卡片 | 微信分享 UI，A 选 **群/朋友圈/文件助手等** | **同一微信群**内点开最容易闭环 |
| 复制码 | A 用 **任意渠道** 把字符串给 B | 短信、其它 IM、电话里念码 |
| 太阳码 | 图片 | 当面扫，或发图到群/私信 |

若产品上要求 **「在好友列表里点 B 的名字，对方就一定能收到」**，在不做 IM、又不保证同群的前提下，**做不到纯前端闭环**，必须 **二期：站内邀请记录 + 对方打开小程序时拉取 / 订阅消息**（见流程 E）。

---

## 五、流程 D：被邀请方 B · 打开并加群

```text
B 点击分享卡片 / 粘贴打开小程序 / 扫码
    │
    ├─ 未登录 ──► 登录（GET /api/auth/wechat/login）
    │
    └─ 落地页带 code ──► POST /api/groups/join-by-invite { invite_code }
            │
            ├─ 成功 ──► Toast「已加入」→ 跳转群详情或「我的群组」
            │
            └─ 失败 ──► 展示后端 message（满员、无效码等）
```

## 六、流程 E（可选迭代）：站内「定向邀请」给好友列表里的 B

**当前后端未做「点对点推送」**。若产品上强依赖「只通知列表里的某个人」，可后续增加：

```text
[好友列表] 某好友右侧「邀请进群」
    │
    └─ 选择群 ──► 服务端创建一条「邀请记录」+ 对方打开小程序时红点/消息列表拉取
                  （需新增接口与前端消息中心）
```

MVP 可先用 **分享卡片 + 复制码**，避免一期做 IM。

## 七、页面与接口对照（便于排期）

| 页面/动作 | 接口 |
|-----------|------|
| 点心加好友 | `POST /api/friends/add` |
| 好友列表 | `GET /api/friends` |
| 创建群 | `POST /api/groups` |
| 群主拉邀请信息 | `GET /api/groups/{id}/invite` |
| 通过码加群 | `POST /api/groups/join-by-invite` |
| 推荐列表（排除已是好友） | `GET /api/match/recommendations` |

---

## 八、文案与合规提示（建议常驻）

1. 「小程序好友」与「微信好友」不同；邀请通过链接/分享完成，无需先加微信。  
2. 涉及微信号展示或一键复制时，遵守隐私与用户授权约定。  
3. 订阅消息若用于「邀请提醒」，单独走微信订阅授权流程，不与爱心混为一谈。
